# 视频分析平台重构说明

## 重构目标

将"生成 Mock 数据"的职责从 Java 后端完全剥离，转交给独立的 Python 脚本，并引入 RabbitMQ 作为通信中间件。

## 数据流向

```
Python 脚本 (模拟算法产生数据) 
    ↓
RabbitMQ 结果队列
    ↓
Spring Boot (监听并转发)
    ↓
WebSocket
    ↓
Vue3 (展示)
```

## 核心业务逻辑

一个完整的"视频分析"任务包含三个子模块，按顺序执行：

1. **音频分析** → 进度 33%
2. **视频分析** → 进度 66%
3. **文本分析** → 进度 100%

## 文件变更清单

### Java 后端

#### 新增文件

1. **`src/main/java/com/ican/project/config/RabbitMQConfig.java`**
   - RabbitMQ 配置类
   - 定义任务队列和结果队列

2. **`src/main/java/com/ican/project/rabbitmq/AlgorithmResultListener.java`**
   - RabbitMQ 消息监听器
   - 监听来自 Python 的消息
   - 转发消息到 WebSocket
   - 保存最终分析结果

#### 修改文件

1. **`pom.xml`**
   - 添加 `spring-boot-starter-amqp` 依赖

2. **`src/main/resources/application.properties`**
   - 添加 RabbitMQ 连接配置

3. **`src/main/java/com/ican/project/service/impl/MockAlgorithmServiceImpl.java`**
   - 删除 `simulateAnalysisProcess()` 方法（不再在 Java 中模拟进度）
   - 修改 `processTask()` 方法，改为发送任务到 RabbitMQ 队列
   - 添加 `RabbitTemplate` 依赖注入

### Python 脚本

#### 新增文件

1. **`algorithm_simulator.py`**
   - 算法模拟脚本
   - 从 RabbitMQ 任务队列获取任务
   - 按顺序执行三个分析模块
   - 发送进度消息到结果队列

2. **`requirements.txt`**
   - Python 依赖：`pika==1.3.2`

### 前端

#### 修改文件

1. **`src/types/index.ts`**
   - 更新 `TaskProgressData` 接口
   - 添加 `moduleType` 和 `resultData` 字段

## 消息格式

### 任务消息（Java -> Python）

```json
{
  "taskId": "任务ID",
  "videoId": "视频ID",
  "videoUrl": "视频URL",
  "videoTitle": "视频标题",
  "videoDuration": 60.0,
  "fileSize": 1024000
}
```

### 进度消息（Python -> Java）

每个阶段完成后发送：

```json
{
  "taskId": "任务ID",
  "moduleType": "音频|视频|文本",
  "progress": 33|66|100,
  "resultData": {
    // 该模块的分析结果
    // 当 progress=100 时，包含所有模块的结果
  }
}
```

## 使用步骤

1. **安装 Python 依赖**
   ```bash
   cd backend
   pip install -r requirements.txt
   ```

2. **启动 RabbitMQ 服务**
   确保 RabbitMQ 在 `192.168.6.130:5672` 运行

3. **启动 Java 后端**
   启动 Spring Boot 应用

4. **启动 Python 脚本**
   ```bash
   cd backend
   python algorithm_simulator.py
   ```

5. **测试**
   - 在前端创建分析任务
   - 观察进度更新（33% → 66% → 100%）
   - 查看分析结果

## 注意事项

1. **队列名称**
   - 任务队列：`algorithm.task.queue`
   - 结果队列：`algorithm.result.queue`

2. **消息持久化**
   - 队列和消息都已配置为持久化

3. **错误处理**
   - Python 脚本处理失败时会发送错误消息
   - Java 端会记录错误日志

4. **并发控制**
   - Python 脚本配置为每次只处理一个任务（prefetch_count=1）

## 测试建议
