# 四步分析流程升级说明

## 变更概述

将分析流程从 **3步** 升级为 **4步**，新增第4步整合分析由Java后端负责。

## 新的分析流程

### 阶段一：音频分析（Python）
- **进度**：25%
- **模块类型**：`audio`
- **负责方**：Python脚本
- **数据流**：Python -> RabbitMQ -> Java -> WebSocket -> Vue3

### 阶段二：视频分析（Python）
- **进度**：50%
- **模块类型**：`video`
- **负责方**：Python脚本
- **数据流**：Python -> RabbitMQ -> Java -> WebSocket -> Vue3

### 阶段三：文本分析（Python）
- **进度**：75%
- **模块类型**：`text`
- **负责方**：Python脚本
- **数据流**：Python -> RabbitMQ -> Java -> WebSocket -> Vue3

### 阶段四：整合分析（Java）
- **进度**：100%
- **模块类型**：`integration`
- **负责方**：Java Spring Boot
- **触发条件**：当Java检测到前三步（audio、video、text）都完成后自动触发
- **生成内容**：受众年龄分布数据
- **数据流**：Java内部处理 -> WebSocket -> Vue3

## 代码修改清单

### Python 脚本 (`algorithm_simulator.py`)

**修改内容：**
1. 调整进度值：25%、50%、75%（原为33%、66%、100%）
2. 修改模块类型标识：使用 `audio`、`video`、`text`（原为中文"音频"、"视频"、"文本"）
3. 移除最终结果合并逻辑：不再发送100%进度，只发送前三步的结果

**关键代码：**
```python
# 第一阶段：音频分析 -> 25%
audio_result = self.simulate_audio_analysis(task_id, task_message)
self.send_progress_message(task_id, "audio", 25, audio_result)

# 第二阶段：视频分析 -> 50%
video_result = self.simulate_video_analysis(task_id, task_message)
self.send_progress_message(task_id, "video", 50, video_result)

# 第三阶段：文本分析 -> 75%
text_result = self.simulate_text_analysis(task_id, task_message)
self.send_progress_message(task_id, "text", 75, text_result)
```

### Java 监听器 (`AlgorithmResultListener.java`)

**新增功能：**
1. **状态跟踪**：使用 `ConcurrentHashMap` 存储每个任务已接收的模块结果
2. **自动触发整合分析**：检测到text模块（75%）完成且所有三个模块都收到后，自动执行第4步
3. **整合分析逻辑**：生成受众年龄分布数据
4. **进度推送**：推送100%进度和整合结果给前端

**关键代码逻辑：**
```java
// 存储当前模块结果
taskModuleResults.computeIfAbsent(taskId, k -> new HashMap<>()).put(moduleType, resultData);

// 检查是否所有三个基础模块都已完成
boolean allModulesCompleted = moduleResults != null 
        && moduleResults.containsKey("audio") 
        && moduleResults.containsKey("video") 
        && moduleResults.containsKey("text");

// 触发整合分析
if (allModulesCompleted && "text".equals(moduleType) && progress == 75) {
    // 执行整合分析
    Map<String, Object> integrationResult = performIntegrationAnalysis(taskId, moduleResults);
    // 推送100%进度
    // ...
}
```

**整合分析生成的数据格式：**
```json
{
  "ageDistribution": {
    "0-18": 0.15,
    "19-35": 0.60,
    "36-50": 0.15,
    "51+": 0.10
  },
  "analysisTimestamp": 1704182400000
}
```

### 前端类型定义 (`frontend/src/types/index.ts`)

**新增类型：**
```typescript
export interface AgeDistribution {
  '0-18': number
  '19-35': number
  '36-50': number
  '51+': number
}

export interface IntegrationResult {
  ageDistribution: AgeDistribution
  analysisTimestamp: number
}
```

## 数据流图

```
用户创建任务
    ↓
Java 发送任务到 RabbitMQ 任务队列
    ↓
Python 获取任务
    ↓
┌─────────────────────────────────────┐
│ Python 执行前三步分析               │
│ 1. 音频分析 (25%) → MQ → Java → WS │
│ 2. 视频分析 (50%) → MQ → Java → WS │
│ 3. 文本分析 (75%) → MQ → Java → WS │
└─────────────────────────────────────┘
    ↓
Java 检测到前三步完成
    ↓
┌─────────────────────────────────────┐
│ Java 执行第4步整合分析              │
│ - 生成受众年龄分布                  │
│ - 更新进度为 100%                   │
│ - 推送结果到 WebSocket              │
└─────────────────────────────────────┘
    ↓
Vue3 前端接收并展示
```

## 测试验证

### 测试步骤
1. 启动 RabbitMQ 服务
2. 启动 Java 后端
3. 启动 Python 脚本：`python algorithm_simulator.py`
4. 在前端创建分析任务
5. 观察进度更新：25% → 50% → 75% → 100%
6. 验证100%时是否收到受众年龄分布数据

### 预期结果
- 前端进度条依次显示：25%、50%、75%、100%
- 每个阶段都能收到对应的模块结果数据
- 100%时收到整合分析结果，包含受众年龄分布
- 受众年龄分布数据格式正确，总和为1.0

## 注意事项

1. **模块类型标识**：Python使用英文标识（audio/video/text），Java会转换为中文显示
2. **状态缓存**：Java使用内存Map存储模块结果，任务完成后会自动清理
3. **消息确认**：所有RabbitMQ消息都需要手动确认，确保消息不丢失
4. **并发安全**：使用 `ConcurrentHashMap` 确保多线程环境下的安全性
