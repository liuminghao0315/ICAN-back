# Python模块化架构说明

## 架构概述

将Python脚本重构为模块化架构，便于团队协作开发。主模块负责RabbitMQ通信和任务协调，视频处理和音频处理分别由不同开发者负责。

## 模块结构

```
backend/
├── algorithm_simulator.py  # 主模块（RabbitMQ消费者，任务协调）
├── 视频处理.py              # 视频处理模块（开发者：潘嘉乐）
└── 音频处理.py              # 音频处理模块（开发者：展鑫鑫）
```

## 模块职责

### 1. algorithm_simulator.py（主模块）

**职责：**
- RabbitMQ连接管理
- 监听任务分发队列
- 模拟媒体分割过程
- 并发调用视频处理和音频处理模块
- 发送结果消息到结果回调队列

**主要方法：**
- `process_task()`: 处理单个分析任务
- `process_visual_stream()`: 调用视频处理模块
- `process_audio_stream()`: 调用音频处理模块
- `send_result_message()`: 发送结果消息到RabbitMQ

### 2. 视频处理.py（视频处理模块）

**开发者：** 潘嘉乐

**职责：**
- 处理视觉流分析
- 分析视频画面内容（场景识别、人脸检测、质量评估等）

**主要函数：**
```python
def process_video(task_id: str, video_info: Dict[str, Any]) -> Dict[str, Any]:
    """
    处理视觉流分析
    
    Args:
        task_id: 任务ID
        video_info: 视频信息（包含 videoUrl, videoTitle, videoDuration, fileSize 等）
        
    Returns:
        视觉流分析结果（JSON格式的字典）

```

**返回数据格式：**
```json
{
  "duration": 60.0,
  "width": 1920,
  "height": 1080,
  "fps": 30,
  "sceneType": "教室",
  "sceneConfidence": 0.9,
  "faceCount": 5,
  "hasPerson": true,
  "qualityScore": 0.85,
  "brightness": 0.6,
  "clarity": 0.8,
  "processingTime": 5.2
}
```

### 3. 音频处理.py（音频处理模块）

**开发者：** 展鑫鑫

**职责：**
- 处理音频流分析
- Step 1: 提取音频特征
- Step 2: ASR转文本

**主要函数：**
```python
def process_audio(task_id: str, video_info: Dict[str, Any]) -> Tuple[Dict[str, Any], Dict[str, Any]]:
    """
    处理音频流分析（包含两个步骤）
    
    Args:

    Returns:
        Tuple[音频特征结果, 文本分析结果]
        第一个字典是音频特征提取结果（moduleType: "audio"）

```

**返回数据格式：**


**文本分析结果（第二个返回值）：**
```json
{
  "transcription": "大家好，今天我来分享一下...",

  "hasDescription": true,
  "titleSentiment": 0.5,
```

## 工作流程

```
1. 主模块收到RabbitMQ任务
   ↓
2. 模拟媒体分割
   ↓
3. 并发启动两个线程：
   ├─ Thread A: 调用 视频处理.process_video()
   │   └─ 返回JSON结果 → 主模块发送 "video" 消息
   │
   └─ Thread B: 调用 音频处理.process_audio()
       ├─ Step 1: 提取音频特征
       │   └─ 返回JSON结果 → 主模块发送 "audio" 消息
       └─ Step 2: ASR转文本
           └─ 返回JSON结果 → 主模块发送 "text" 消息
```

## 消息发送流程

### 视频处理流程
```
主模块 → 调用 视频处理.process_video()
      → 接收返回的JSON结果
      → 发送 "video" 类型消息到RabbitMQ
```

### 音频处理流程
```
主模块 → 调用 音频处理.process_audio()
      → 接收返回的Tuple[audio_result, text_result]
      → 发送 "audio" 类型消息到RabbitMQ（使用audio_result）
      → 发送 "text" 类型消息到RabbitMQ（使用text_result）
```

## 消息格式

所有消息都通过主模块的 `send_result_message()` 方法发送：

```json
{
  "taskId": "任务ID",
  "moduleType": "video|audio|text",
  "resultData": {
    // 模块特定的分析结果
  }
}
```

## 开发指南

### 对于潘嘉乐（视频处理模块）

1. **修改文件：** `视频处理.py`
2. **修改函数：** `process_video()`
3. **输入参数：**
   - `task_id`: 任务ID（字符串）
   - `video_info`: 视频信息字典，包含：
     - `videoUrl`: 视频URL
     - `videoTitle`: 视频标题
     - `videoDuration`: 视频时长（秒）
     - `fileSize`: 文件大小（字节）
4. **返回要求：**
   - 返回一个字典（Dict[str, Any]）
   - 字典内容会被主模块包装后发送到RabbitMQ
5. **注意事项：**
   - 函数内部可以添加日志（使用 `logger`）
   - 可以修改处理逻辑，但不要改变函数签名
   - 返回的字典会被JSON序列化，确保所有值都是可序列化的

### 对于展鑫鑫（音频处理模块）

1. **修改文件：** `音频处理.py`
2. **修改函数：** 
   - `process_audio()`: 主函数，返回两个结果
   - `extract_audio_features()`: 音频特征提取（可选修改）
   - `convert_audio_to_text()`: ASR转文本（可选修改）
3. **输入参数：**
   - `task_id`: 任务ID（字符串）
   - `video_info`: 视频信息字典（同上）
4. **返回要求：**
   - 返回一个元组：`Tuple[Dict[str, Any], Dict[str, Any]]`
   - 第一个字典是音频特征结果（moduleType: "audio"）
   - 第二个字典是文本分析结果（moduleType: "text"）
5. **注意事项：**
   - 函数内部可以添加日志（使用 `logger`）
   - 可以修改处理逻辑，但不要改变函数签名
   - 两个步骤是顺序执行的（Step 1完成后才执行Step 2）
   - 返回的字典会被JSON序列化，确保所有值都是可序列化的

## 错误处理

### 模块内部错误

如果子模块抛出异常：
- 主模块会捕获异常
- 记录错误日志
- 发送错误消息到RabbitMQ（moduleType: "error"）
- 不会影响另一个模块的执行

### 消息发送错误

如果消息发送失败：
- 记录错误日志
- 抛出异常（会被RabbitMQ重试机制处理）

## 协作注意事项

1. **接口约定：**
   - 不要修改函数签名（参数和返回值）
   - 确保返回的数据格式符合要求

2. **日志规范：**
   - 使用 `logger.info()` 记录关键步骤
   - 使用 `logger.error()` 记录错误
   - 日志格式：`[任务 {task_id}] [模块名] 消息内容`

3. **代码提交：**
   - 只修改自己负责的模块文件
   - 提交前确保代码可以正常运行
   - 不要修改主模块的RabbitMQ相关代码

4. **测试建议：**
   - 修改代码后先本地测试
   - 确保返回的数据格式正确
   - 检查日志输出是否正常
