# 整合分析统计算法说明

## 概述

第四步整合分析不再使用随机模拟数据，而是基于前三个模块（video、audio、text）的真实数据，通过加权统计算法计算出最终的受众分析结果。

## 算法架构

```
输入：video模块结果 + audio模块结果 + text模块结果
  ↓
加权统计算法
  ├─ 受众年龄分布计算
  ├─ 预测兴趣标签计算
  ├─ 预测播放量计算
  └─ 预测互动率计算
  ↓
输出：整合分析结果
```

## 1. 受众年龄分布计算算法

### 输入特征
- **场景类型**（video.sceneType）：教室、图书馆、操场、宿舍、食堂、实验室、报告厅、校园户外
- **场景置信度**（video.sceneConfidence）：0-1
- **主题分类**（text.topicCategory）：校园生活、学术讨论、社团活动等
- **情感评分**（text.sentimentScore）：-1到1
- **视频质量**（video.qualityScore）：0-1

### 算法逻辑

#### 1.1 初始化基础权重
```java
18-24岁: 25% (基础权重)
25-34岁: 35% (基础权重)
35-44岁: 25% (基础权重)
45+岁:   15% (基础权重)
```

#### 1.2 场景类型调整（置信度>0.8时生效）
- **学术场景**（教室、图书馆、实验室）：
  - 18-24岁 +15%
  - 25-34岁 +5%
  
- **生活场景**（操场、宿舍、食堂）：
  - 18-24岁 +10%
  - 25-34岁 +10%
  
- **正式场景**（报告厅）：
  - 25-34岁 +10%
  - 35-44岁 +10%

#### 1.3 主题分类调整
- **校园主题**（校园生活、社团活动、体育运动）：
  - 18-24岁 +12%
  - 25-34岁 +5%
  
- **学术主题**（学术讨论、科技创新）：
  - 18-24岁 +8%
  - 25-34岁 +12%
  
- **职业主题**（就业指导、创业分享）：
  - 25-34岁 +15%
  - 35-44岁 +5%
  
- **社会主题**（心理健康、社会实践）：
  - 25-34岁 +8%
  - 35-44岁 +8%

#### 1.4 情感评分调整
- **正面情感**（sentimentScore > 0.3）：
  - 18-24岁 +8%
  - 25-34岁 +5%
  
- **负面情感**（sentimentScore < -0.3）：
  - 35-44岁 +5%
  - 45+岁 +5%

#### 1.5 视频质量调整
- **高质量视频**（qualityScore > 0.8）：
  - 18-24岁 +5%
  - 25-34岁 +3%

#### 1.6 归一化
所有权重相加后归一化，确保总和为100%。

### 输出
```json
{
  "18-24": 0.35,  // 35%
  "25-34": 0.45,  // 45%
  "35-44": 0.15,  // 15%
  "45+": 0.05     // 5%
}
```

## 3. 预测播放量计算算法

### 输入特征
- **视频质量**（video.qualityScore, clarity, brightness）
- **音频质量**（audio.audioQuality, speechRatio, noiseLevel）
- **情感评分**（text.sentimentScore）
- **主题分类**（text.topicCategory）
- **视频时长**（video.duration）
- **人员出现**（video.hasPerson）

### 算法逻辑

#### 3.1 基础播放量
```
baseViews = 100 + random(0-400) = 100-500
```

#### 3.2 加权因子计算

**视频质量因子（权重30%）**
```
videoQualityFactor = (qualityScore * 0.5 + clarity * 0.3 + brightness * 0.2) * 0.3
```

**音频质量因子（权重20%）**
```
audioQualityFactor = (audioQuality * 0.6 + speechRatio * 0.3 + (1 - noiseLevel) * 0.1) * 0.2
```

**情感因子（权重25%）**
```
sentimentFactor = ((sentimentScore + 1) / 2) * 0.25
// 正面情感播放量更高
```

**主题热度因子（权重15%）**
```
topicHeatFactor = getTopicHeatFactor(topicCategory) * 0.15

主题热度映射：
- 校园生活、社团活动: 0.9 (高热度)
- 学术讨论、科技创新: 0.8
- 就业指导、创业分享: 0.85
- 体育运动、艺术表演: 0.75
- 心理健康、社会实践: 0.7
- 其他: 0.65
```

**视频时长因子（权重10%）**
```
durationFactor = calculateDurationFactor(duration) * 0.1

时长因子映射：
- < 60秒: 0.3 (太短)
- 120-300秒 (2-5分钟): 1.0 (最佳)
- 300-600秒 (5-10分钟): 0.7
- > 600秒 (>10分钟): 0.4 (太长)
```

**人员出现因子（权重5%）**
```
personFactor = hasPerson ? 0.05 : 0.02
```

#### 3.3 总因子计算
```
totalFactor = videoQualityFactor + audioQualityFactor + sentimentFactor + 
              topicHeatFactor + durationFactor + personFactor
```

#### 3.4 最终播放量计算
```
predictedViews = baseViews * (1 + totalFactor * 20)
// 倍数范围：1-21倍
// 限制范围：100-50000
```

### 输出
```json
{
  "predictedViews": 3500  // 100-50000之间的整数
}
```

## 4. 预测互动率计算算法

### 输入特征
- **情感评分**（text.sentimentScore）
- **内容质量**（video.qualityScore, clarity）
- **语音比例**（audio.speechRatio）
- **人员出现**（video.hasPerson）
- **关键词数量**（text.keywords.length）

### 算法逻辑

#### 4.1 基础互动率
```
baseEngagement = 0.01 + random(0-0.02) = 1%-3%
```

#### 4.2 加权因子计算

**情感因子（权重30%）**
```
sentimentFactor = |sentimentScore| * 0.3
// 极端情感（正面或负面）互动率更高
```

**内容质量因子（权重25%）**
```
contentQualityFactor = (qualityScore * 0.6 + clarity * 0.4) * 0.25
```

**语音比例因子（权重20%）**
```
speechFactor = speechRatio * 0.2
// 有语音内容的视频互动率更高
```

**人员出现因子（权重15%）**
```
personFactor = (hasPerson ? 0.8 : 0.3) * 0.15
```

**关键词丰富度因子（权重10%）**
```
keywordFactor = min(1.0, keywordCount / 5.0) * 0.1
// 最多5个关键词为满分
```

#### 4.3 总因子计算
```
totalFactor = sentimentFactor + contentQualityFactor + speechFactor + 
              personFactor + keywordFactor
```

#### 4.4 最终互动率计算
```
predictedEngagement = baseEngagement * (1 + totalFactor * 2)
// 倍数范围：1-3倍
// 限制范围：0.5%-15%
```

### 输出
```json
{
  "predictedEngagement": 0.045  // 4.5%
}
```

## 算法特点

### 1. 多维度加权
- 综合考虑视频、音频、文本三个模块的特征
- 不同特征使用不同权重，反映其重要性

### 2. 区间计算
- 年龄分布基于场景、主题、情感等多维度特征进行区间调整
- 播放量和互动率基于多个因子加权计算

### 3. 归一化处理
- 年龄分布确保总和为100%
- 所有因子归一化到0-1区间

### 4. 边界限制
- 播放量限制在100-50000之间
- 互动率限制在0.5%-15%之间
- 防止极端值出现

### 5. 容错处理
- 数据缺失时使用默认值
- 类型转换异常时使用安全方法

## 使用示例

### 输入数据
```json
{
  "video": {
    "sceneType": "教室",
    "sceneConfidence": 0.9,
    "qualityScore": 0.85,
    "clarity": 0.8,
    "brightness": 0.6,
    "duration": 180,
    "hasPerson": true
  },
  "audio": {
    "audioQuality": 0.9,
    "speechRatio": 0.7,
    "noiseLevel": 0.1
  },
  "text": {
    "topicCategory": "学术讨论",
    "sentimentScore": 0.6,
    "keywords": ["学习", "考试", "考研"]
  }
}
```

### 计算过程
1. **年龄分布**：
   - 场景：教室（学术场景）→ 18-24岁+15%
   - 主题：学术讨论 → 18-24岁+8%, 25-34岁+12%
   - 情感：正面 → 18-24岁+8%
   - 质量：高质量 → 18-24岁+5%
   - 最终：18-24岁≈45%, 25-34岁≈50%, 其他≈5%

2. **兴趣标签**：
   - 主题：学术讨论 → ["教育", "学术研究"]
   - 关键词：学习、考试、考研 → ["教育"]
   - 场景：教室 → ["学术研究"]
   - 最终：["教育", "学术研究"]

3. **播放量**：
   - 视频质量因子：0.85*0.5+0.8*0.3+0.6*0.2 = 0.785 → 0.2355
   - 音频质量因子：0.9*0.6+0.7*0.3+0.9*0.1 = 0.84 → 0.168
   - 情感因子：(0.6+1)/2 = 0.8 → 0.2
   - 主题热度：0.8 → 0.12
   - 时长因子：180秒在120-300区间 → 1.0 → 0.1
   - 人员因子：有人员 → 0.05
   - 总因子：0.8735
   - 基础播放量：300
   - 最终：300 * (1 + 0.8735 * 20) = 300 * 18.47 = 5541

4. **互动率**：
   - 情感因子：|0.6| * 0.3 = 0.18
   - 内容质量：0.85*0.6+0.8*0.4 = 0.83 → 0.2075
   - 语音比例：0.7 * 0.2 = 0.14
   - 人员因子：0.8 * 0.15 = 0.12
   - 关键词因子：3/5 = 0.6 → 0.06
   - 总因子：0.7075
   - 基础互动率：0.02
   - 最终：0.02 * (1 + 0.7075 * 2) = 0.02 * 2.415 = 0.0483 (4.83%)

## 算法优势

1. **真实性**：基于实际分析数据，而非随机生成
2. **可解释性**：每个计算结果都有明确的特征依据
3. **可调优**：权重和映射关系可以根据实际数据调整
4. **鲁棒性**：容错处理确保数据缺失时仍能计算
5. **合理性**：边界限制和归一化确保结果在合理范围内
