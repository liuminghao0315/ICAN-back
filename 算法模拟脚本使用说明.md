# 算法模拟脚本使用说明

## 架构说明

新的架构将算法模拟逻辑从 Java 后端剥离到独立的 Python 脚本，通过 RabbitMQ 进行通信：

```
用户创建任务 -> Java 后端 -> RabbitMQ 任务队列 -> Python 脚本
                                                      ↓
Python 脚本处理（音频33% -> 视频66% -> 文本100%）
                                                      ↓
RabbitMQ 结果队列 -> Java 后端 -> WebSocket -> Vue3 前端
```

## 环境要求

### Java 后端
- Spring Boot 3.0.2
- RabbitMQ 依赖已添加到 `pom.xml`

### Python 脚本
- Python 3.7+
- 依赖包：`pika`（RabbitMQ 客户端）

## 配置说明

### RabbitMQ 配置

在 `application.properties` 中已配置：
```properties
spring.rabbitmq.host=192.168.6.130
spring.rabbitmq.port=5672
spring.rabbitmq.username=guest
spring.rabbitmq.password=guest
spring.rabbitmq.virtual-host=/
```

### Python 脚本配置

在 `algorithm_simulator.py` 中配置：
```python
RABBITMQ_HOST = '192.168.6.130'
RABBITMQ_PORT = 5672
RABBITMQ_USERNAME = 'guest'
RABBITMQ_PASSWORD = 'guest'
```

## 使用步骤

### 1. 安装 Python 依赖

```bash
cd backend
pip install -r requirements.txt
```

### 2. 启动 RabbitMQ 服务

确保 RabbitMQ 服务正在运行（默认端口 5672）。

### 3. 启动 Java 后端

启动 Spring Boot 应用。

### 4. 启动 Python 脚本

```bash
cd backend
python algorithm_simulator.py
```

脚本会持续运行，监听任务队列。

## 工作流程

1. **任务创建**：用户在前端创建分析任务
2. **任务发送**：Java 后端将任务信息发送到 `algorithm.task.queue` 队列
3. **任务处理**：Python 脚本从队列获取任务，按顺序执行：
   - 音频分析（2-5秒）→ 进度 33%
   - 视频分析（3-6秒）→ 进度 66%
   - 文本分析（2-4秒）→ 进度 100%
4. **进度推送**：每个阶段完成后，Python 发送进度消息到 `algorithm.result.queue`
5. **消息转发**：Java 后端监听结果队列，将消息转发到 WebSocket
6. **前端展示**：Vue3 前端通过 WebSocket 接收并展示进度

## 消息格式

### 任务消息（Java -> Python）

```json
{
  "taskId": "任务ID",
  "videoId": "视频ID",
  "videoUrl": "视频URL",
  "videoTitle": "视频标题",
  "videoDuration": 60.0,
  "fileSize": 1024000
}
```

### 进度消息（Python -> Java）

```json
{
  "taskId": "任务ID",
  "moduleType": "音频|视频|文本",
  "progress": 33|66|100,
  "resultData": {
    "audioResult": { ... },
    "videoResult": { ... },
    "textResult": { ... },
    "riskScore": 0.5,
    "riskLevel": "LOW|MEDIUM|HIGH",
    ...
  }
}
```

## 注意事项

1. **队列持久化**：队列已配置为持久化，消息不会因服务重启而丢失
2. **消息确认**：Python 脚本处理完任务后会确认消息，确保消息不会重复处理
3. **错误处理**：如果处理失败，Python 会发送错误消息到结果队列
4. **并发处理**：Python 脚本配置为每次只处理一个任务（prefetch_count=1）

## 故障排查

### Python 脚本无法连接 RabbitMQ
- 检查 RabbitMQ 服务是否运行
- 检查网络连接和防火墙设置
- 验证用户名密码是否正确

### 任务没有被处理
- 检查 Python 脚本是否正在运行
- 检查队列中是否有消息堆积
- 查看 Python 脚本日志

### 前端没有收到进度更新
- 检查 WebSocket 连接是否正常
- 检查 Java 后端日志，确认是否收到 RabbitMQ 消息
- 检查用户 ID 是否正确
